# 总结

## 浏览器

### 概述

网页的渲染流程简单来说就是: `URL`输入 --- 请求得到`HTML` --- 浏览器将`HTML`解析转换成`DOM` --- 再将`css`加在对应的`DOM node`上 --- 然后进行`layout` --- 最终生成`BitMap` 

### 网络请求

##### - `HTTP`请求和响应

```
请求

URL输入 -> 根据域名解析IP地址 -> 建立TCP链接 -> 发送HTTP请求

浏览器通过URL地址向指定的服务器请求资源，在请求报文中携带信息可以分为三个部分：请求行、请求头、请求数据
	请求行:
		由请求方法、请求地址、HTTP协议版本三个部分组成
	请求头:
		请求头为请求报文添加一些附加的信息，由`名:值`对组成，最终以空行结尾
	请求数据:
		以等号赋值的形式通过&符号拼接而成的字符串

```

```
响应

HTTP响应报文由状态行（status line）、相应头部（headers）、空行（blank line）和响应数据（response body）4个部分组成。
	状态行:
		状态行由3部分组成，分别为：协议版本、状态码、状态码
	相应头部:
		与请求头相同
	响应数据:
		用于存放需要返回给客户端的数据信息。
```



### `HTML`解析

##### -  `HTML`解析流程

```
词法分析过程，将输入内容解析成多个标记

包括起始标记，结束标记，属性名称，属性值

标记生成器识别标记，传递给树构造器，然后接受下一个字符以识别下一个标记,如此反复直到输入的结束。标记化算法的输出结果是html标记
```



##### - `css` 样式添加对应的`DOM node`上

```
选择器匹配
	通过HTML的属性，和标签间的关系匹配对应的样式

样式的权重
	如果存在相同的样式属性则通过比较两个样式的选择器的权重，决定实际样式
	内联 -> id -> class -> 标签
```



##### - `layout`

```
通过样式 -> 排版、布局相关样式处理 -> 标签的大小、位置 -> 布局 -> 生成图层树
```

##### - 生成图片

```
最终根据layout生成的图层树，叠加后合成了最终的页面
```
